1.    機能

SMBSyncはAndroid端末とPCやNASとの間でファイルの同期を行うためのツールです。同期の方法としてミラー、移動、コピーの３種類が使用できます。

Android端末の内部ストレージとSDCARD間での同期も可能です。

        ミラー

マスター側のディレクトリーとファイルをターゲット側に差分コピーし、コピー終了後にマスター側に存在しないターゲット側のファイルとディレクトリーを削除する。

        移動

マスター側のディレクトリーとファイルをターゲット側にコピーし、ターゲット側にコピーしたマスター側のファイルを削除する。（ただし、マスターとターゲットに同名でファイルサイズと更新日時が同じファイルはコピーせずマスター側のファイルを削除）

        コピー

マスター側のディレクトリーに含まれるファイルをターゲット側に差分コピーする。（ただし、マスターとターゲットに同名でファイルサイズと更新日時が同じ場合はコピーしない）

下記の３条件のうちいずれかが成立した場合に差分ファイルと判定。

        ファイルが存在しない

        ファイルサイズが違う

        最終更新日時が3秒以上違う(秒数はオプションにより変更可能)

目次に戻る

2.    使用方法

ミラーと移動はファイルが削除されます、コピーはファイルを上書きしますので、テストディレクトリーを作成してテストしてからご使用ください。

同期を行うにはローカル、リモート、同期のプロファイルを作成することが必要です。ローカルからローカルへの同期（内部ストレージからUSBストレージなど）ではリモートプロファイルは不要です。

        ローカルプロファイルはローカル側（Android側）のディレクトリーを指定します。

        リモートプロファイルはリモート側（NASやPCなどのSMBサーバー）のディレクトリーを指定します。

        同期プロファイルはローカルまたはリモートのプロファイルをマスターまたはターゲットとして指定し、同期タイプを指定します。

同期は同期プロファイル毎に行います。同期する順序はプロファイルタブに表示されている順番です。（同期プロファイルの名前の昇順になります）

プロファイルの作成例

 

上記の例ではWiFiで接続されたAndroid端末にPCとNASのフォルダーからファイルを同期するプロファイルを示している。

        SYNC1はPCのフォルダーからLocalの/mnt/sdcard/DCIM/DIR-PCにjpgファイルをミラーで同期をおこないます。

        SYNC2はNASのフォルダーからLocalの/mnt/sdcard/DCIM/DIR-NASにjpgファイルをミラーで同期をおこないます。

2.3.3 同期プロファイルの作成

コンテキストメニューから同期プロファイルの追加をクリック。

＊１：ボタンをタップするとプロファイルを編集できます。

        有効

プロファイルを有効にする

        プロファイル名

プロファイルの名前を指定する

        マスタープロファイル

同期の元となるプロファイルを指定する。左端のボタンを押すことにより選択されたプロファイルを編集できます。

        ターゲットプロファイル

同期先となるプロファイルを指定する。左端のボタンを押すことにより選択されたプロファイルを編集できます。

マスターとターゲットプロファイルはリモートとローカルまたはローカルとローカルの組み合わせにすることが必要です。たとえばマスターがリモートならばターゲットはローカルという組み合わせにしてください。

        同期方式

ミラー、コピーと移動の方式を選択

        フィルター（ファイル）

ファイル選択フィルターを参照

        フィルター（ディレクトリー）

ディレクトリー選択フィルターを参照

        マスターディレクトリーのファイルとディレクトリーを処理

チェックした場合はマスタープロファイルで指定されたディレクトリーに存在するファイルとディレクトリーを処理します。（省略時値はチェック有りです）

チェックしていない場合はマスタープロファイルで指定されたディレクトリーに存在ディレクトリーだけを処理します。

        上書コピーまたは削除の前に確認する

チェックした場合はファイルの削除と上書きするときに確認ダイアログを表示します。

        リモートファイルの最終更新時刻をローカルファイルに合わせない

通常は使用しない。

リモートの制限によりSmbFile#setLastModified()が失敗する場合に有効にしてください。

これにより、ローカルとリモートのファイルの差分判定はファイルの存在の有無とファイルサイズで判定されます。

        強制的にSMBSyncで最終更新日時を取得

通常は使用しない。

Java File#setLastModified()を使用しないで、SMBSync独自方式でローカルファイルの最終更新日時を保守する。

        無線LANがオフでも同期を実行可能にする

        無線LANがオンでアクセスポイントに接続していなくても同期を実行可能にする

        無線LANがオンでアクセスポイントに接続しているときに同期を実行可能にする

        自動同期起動

チェックされている場合は、アプリケーション起動後に有効なプロファイル全ての同期を実行します。

        自動停止（自動同期起動が選択されている場合に有効）

チェックされている場合は、同期が終了したらアプリケーションを終了します。

        バックグラウンド実行（自動同期起動が選択されている場合に有効）

チェックされている場合は、ホームスクリーン(バックグランド実行)を表示します。

        エラー時のオプション

チェックされている場合は、同期実行時にエラーが発生しても無視します。

        同期中の画面オン

チェックされている場合は、同期実行中に画面消灯を許容しない

        リモートコピー先ファイルの保存

チェックされている場合は、リモートへのファイルコピー時にコピー完了までコピー先のファイルを保存する(コピーが中止された場合はファイルの内容は変更されません)

        ローカルでのコピー先ファイルの保存

チェックされている場合は、ローカルへのファイルコピー時にコピー完了までコピー先のファイルを保存する(コピーが中止された場合はファイルの内容は変更されません)

4.8.2 ユーザーインターフェース

        代替UI

「他の項目が選択されていないときにクリックしたら編集、他の項目が選択されているときは項目を選択」を有効にする

        警告メッセージ抑止

マウントポイント混在の警告メッセージを抑止するかどうかを決めます。

        デバッグメッセージの表示

デバッグメッセージをメッセージタブに表示するかどうかを決めます。

        バックグラウンド終了の通知

バックグラウンド実行状態でアプリケーションが終了した場合（自動終了が有効な場合）のNotificationでの通知方法を選択します。

バックグラウンドで実行中は必ずNotificationに通知され、状態が確認できます。

        同期終了時の通知音の再生

同期終了時に省略時通知音を再生するかどうかを設定します。

        同期終了時のバイブレーション

同期終了時に振動させるかどうかを設定します。

 

4.8.3 ログ

        ログ記録

ログメッセージをファイルに保存する方法を決めます。

        作成しない

        ファイル名に日付(log-yyyy_mm_dd.txt)をつけて、追記（ファイルが存在しない場合は作成します)

        ログファイルの保存世代

        ログ

ログメッセージに書き出す種類を決めます。

        ログファイルディレクトリ

ログメッセージファイルのディレクトリーを決めます。(初期設定は/mnt/sdcard/SMBSync)

4.8.4 MediaScannerオプション

        非メディアファイル

非メディアファイルをコピー終了時にMediaScannerでスキャンするかどうかを指定する

        外部ストレージ

USBやSDCARD等の外部メディアへのコピー終了時にMediaScannerでスキャンするかどうかを指定する

4.8.5 差分ファイル判定

        最終更新時刻の差

差分ファイルとするリモートとローカル ファイルの最終更新時刻の差を秒数で指定します。（初期値は３秒）

        メディアストアファイルの最終更新時刻

メディアストア（DCIMやMusicフォルダー等）のファイルについて、ファイル最終更新時刻の取り扱いを選択します。

        最終更新時刻だけで差分を判定

        最終更新時刻と最終更新時刻＋タイムゾーンで差分を判定

        ファイル最終更新時刻を使用しない

端末再起動によるメディアストアフォルダーのファイルの最終更新時間が書き換わりファイルが毎回コピーされる場合は（２）を試してください。

        ＳＭＢパフォーマンス

5.2 サンプルコード

下記のコードはActivityから起動するサンプルです。

 

public class MainActivity extends Activity {

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.activity_main);

}

@Override

protected void onResume() {

super.onResume();

 

PackageManager pm = getPackageManager();

Intent in=pm.getLaunchIntentForPackage("com.sentaroh.android.SMBSync");

in.setAction(Intent.ACTION_MAIN);

in.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK);

in.putExtra("AutoStart", true);

in.putExtra("AutoTerm", true);

in.putExtra("Background", true);

String[] prof=new String[] {"S-PGMS","S-Pictures"};

in.putExtra("SyncProfile", prof);

startActivity(in);

 

}

}

6.    サンプルプロファイルについて
